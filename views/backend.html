Back-End Development.

Back end development is alle digitale handelingen achter de schermen die nodig zijn om de voorkant van een website draaiende te houden, zoals de codering, stijl en plug-ins. De voorkant van uw website is wat het publiek kan ziet, omvat de achterkant de stagehands, visagisten, klanten, technische crew, toneelmanagers, enz.

Termen in Back End Development.

API:

Een Application Programming Interface (API) is een verbinding tussen computers of tussen computerprogramma's. Het is een soort software-interface die een dienst aanbiedt aan andere stukjes software. Een document of standaard die beschrijft hoe een dergelijke verbinding of interface moet worden gebouwd of gebruikt, wordt een API-specificatie genoemd. Een computersysteem dat aan deze norm voldoet, zou een API implementeren of blootleggen. De term API kan verwijzen naar de specificatie of naar de implementatie.

In tegenstelling tot een gebruikersinterface, die een computer met een persoon verbindt, verbindt een applicatieprogrammeerinterface computers of stukjes software met elkaar. Het is niet bedoeld om rechtstreeks te worden gebruikt door een andere persoon (de eindgebruiker) dan een computerprogrammeur die het in software integreert. Een API bestaat vaak uit verschillende onderdelen die fungeren als tools of services die beschikbaar zijn voor de programmeur. Een programma of een programmeur die een van deze onderdelen gebruikt, zou dat deel van de API aanroepen. De aanroepen waaruit de API bestaat, worden ook wel subroutines, methoden, verzoeken of eindpunten genoemd. Een API-specificatie definieert deze aanroepen, wat betekent dat wordt uitgelegd hoe ze te gebruiken of te implementeren.

Een doel van API's is om de interne details van hoe een systeem werkt te verbergen, alleen die delen bloot te leggen die een programmeur nuttig vindt en ze consistent te houden, zelfs als de interne details later veranderen. Een API kan op maat worden gemaakt voor een bepaald paar systemen, of het kan een gedeelde standaard zijn die interoperabiliteit tussen veel systemen mogelijk maakt.

Firewall:

Firewalls analyseren binnenkomend verkeer zorgvuldig op basis van vooraf vastgestelde regels en filteren verkeer afkomstig van onbeveiligde of verdachte bronnen om aanvallen te voorkomen. Firewalls bewaken het verkeer op het toegangspunt van een computer, poorten genoemd, waar informatie wordt uitgewisseld met externe apparaten.

Bijvoorbeeld: "Bronadres 172.18.1.1 mag bestemming 172.18.2.1 bereiken via poort 22."

Denk aan IP-adressen als huizen en poortnummers als kamers in het huis. Alleen vertrouwde mensen (bronadressen) mogen het huis (bestemmingsadres) überhaupt betreden - dan wordt het verder gefilterd zodat mensen in het huis alleen toegang hebben tot bepaalde kamers (bestemmingspoorten), afhankelijk van of ze de eigenaar zijn , een kind of een gast. De eigenaar heeft toegang tot elke kamer (elke poort), terwijl kinderen en gasten een bepaalde reeks kamers mogen betreden (specifieke poorten).

Data Base:

Een database is een systematische verzameling van gegevens. Ze ondersteunen elektronische opslag en manipulatie van gegevens. Databases maken databeheer eenvoudig.

Laten we een voorbeeld van een database bespreken: Een online telefoongids gebruikt een database om gegevens van mensen, telefoonnummers en andere contactgegevens op te slaan. Uw elektriciteitsleverancier gebruikt een database om facturering, klantgerelateerde problemen te beheren, storingsgegevens te verwerken, enz.

Laten we ook eens kijken naar Facebook. Het moet gegevens opslaan, manipuleren en presenteren met betrekking tot leden, hun vrienden, ledenactiviteiten, berichten, advertenties en nog veel meer. Voor het gebruik van databases kunnen we talloze voorbeelden geven.
